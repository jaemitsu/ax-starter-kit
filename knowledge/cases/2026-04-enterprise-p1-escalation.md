# 사례: 엔터프라이즈 P1 장애 티켓 에스컬레이션 지연 (2026-04)

## 상황
엔터프라이즈 D고객의 로그인 불가 장애가 P3으로 오분류되어 최초 응답까지 5시간 소요.
고객이 계약 해지를 언급하며 임원 채널로 항의.

## 원인
챗 채널 접수 시 "로그인이 가끔 안 돼요"라는 표현을 트리아지가 일반 문의로 분류.
실제로는 SSO 연동 전체 장애였고, 24시간 내 동일 증상 티켓이 4건 있었으나 상관관계를 보지 못했다.

## 해결
P1 재분류 → EscalationCase 오픈 → CX 리드가 엔지니어링 소집 → SSO 인증서 만료 확인 후 2시간 내 복구.
고객에게는 CSM이 타임라인 전체를 공유하고 재발 방지책을 문서로 전달.

## 교훈
- 엔터프라이즈 고객의 인증·접속 관련 문의는 증상 표현과 무관하게 P2 이상으로 시작할 것
- 동일 증상 티켓 다건 발생은 장애 신호 — 24시간 내 3건 이상이면 자동 에스컬레이션 (정책에 반영됨)
- 트리아지 오분류가 SLA 위반의 최대 원인. 재분류 이력을 남겨 트리아지 규칙을 개선할 것
