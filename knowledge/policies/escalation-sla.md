# 에스컬레이션 SLA 정책

## 우선순위별 기한
| 우선순위 | 최초 응답 | 해결 목표 |
|---|---|---|
| P1 (서비스 불가) | 30분 | 4시간 |
| P2 (핵심 기능 장애) | 2시간 | 1영업일 |
| P3 (일반 문의) | 8시간 | 3영업일 |
| P4 (개선 요청) | 1영업일 | 목표 없음 |

기한의 정본은 SLAPolicy 객체다. 이 표는 요약이며 불일치 시 온톨로지가 우선한다.

## 자동 에스컬레이션
P1 티켓이 30분 내 최초 응답되지 않으면 EscalationCase가 자동 오픈되고 CX 리드에게 라우팅된다 (규칙 R-04).
엔터프라이즈 고객의 P1/P2는 담당 CSM에게 동시 통보한다.

## 상위 개입 기준
다음 신호 중 하나라도 있으면 상담원은 즉시 에스컬레이션해야 한다:
- 해결 목표 기한의 50% 경과 시점에 원인 미확인
- 고객이 계약 해지·환불을 언급
- 동일 문제 티켓이 24시간 내 3건 이상 접수 (장애 의심)

## 근거
SLA 정책 v3 (2026-06 개정), 엔터프라이즈 표준 계약서 지원 조항.
